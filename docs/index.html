Privacy Policy

Bulk PDF Downloader Last updated: May 14, 2026

TL;DR: This extension collects no personal data, sends nothing to any server, and runs entirely inside your browser. It only reads the URLs and titles of your open tabs to find PDFs to download, and saves a single on/off toggle locally.

1. Who we are

"Bulk PDF Downloader" (the "Extension") is an open-source Chrome browser extension that detects open tabs whose URLs end with .pdf and lets the user download them in bulk with a single click. The Extension is developed and maintained by the project owner via the public repository at github.com/Mei0001/all-pdf-downloader .

2. What information we collect

None. The Extension does not collect, store remotely, or transmit any of the following:

  • Personally identifiable information (name, email, phone, address, age, ID numbers)
  • Health information
  • Financial or payment information
  • Authentication information (passwords, cookies, OAuth tokens, security questions)
  • Personal communications (emails, chat messages)
  • Location information (region, IP address, GPS coordinates)
  • Web browsing history
  • User activity (clicks, scrolls, mouse position, keystrokes)
  • Website content (text, images, audio, video, hyperlinks)

3. What the Extension does access

When the user opens the Extension's popup, the Extension calls chrome.tabs.query() to retrieve the URL and title of currently open tabs. This information is used in-memory only for the following purposes:

  • To detect tabs whose URL pathname ends with .pdf
  • To display the list of detected PDF tabs in the popup so the user can confirm
  • To use the tab title as the suggested filename when downloading

This data is not persisted to local storage, not transmitted to any external server, and is discarded from memory when the popup is closed.

4. Permissions and why we need them

PermissionPurpose
tabs Read URLs and titles of open tabs to detect PDFs and use the title as the saved filename. Tab contents (DOM, cookies, form inputs) are not accessed.
downloads Save detected PDF files to the user's default Downloads folder. Triggered only by an explicit click of the "Download all" button.
storage Persist a single boolean preference (closeAfterDownload) using chrome.storage.local. Local only — not synced to Google account, not transmitted anywhere.

5. Data we store locally

The only data stored on disk by the Extension is the following preference, kept in chrome.storage.local:

  • closeAfterDownload: a boolean indicating whether tabs should be closed automatically after their PDF finishes downloading.

This preference is stored on the user's local machine only. It is not synchronized across devices and is not transmitted off the device. Uninstalling the Extension removes this preference along with the Extension itself.

6. Third-party services

The Extension does not integrate with any third-party services, analytics platforms, advertising networks, error reporting services, or external APIs. There is no background communication of any kind.

7. Remote code

The Extension does not load, execute, or evaluate remote code. All JavaScript is bundled inside the extension package. No eval(), no remote <script> tags, no dynamic module loading from external sources.

8. Children's privacy

The Extension does not knowingly collect data from anyone, including children under 13. There are no accounts, no sign-up, and no profiles to populate.

9. Changes to this policy

If this policy is updated, the "Last updated" date at the top of this page will reflect the change. Material changes will be announced via the GitHub repository's release notes.
